## 核心技术债务（按优先级）

### P0 — 理论缺失（影响断卦准确性）

1. **三合局未介入吉凶判断**  
   `find_san_he_ju()` 已检测到三合局，但 `judge_dong_gua()` 中三合局的生世/克世逻辑从未被调用。

2. **伏神取用未实现**  
   用神不上卦时无法通过伏神获取，当前直接返回"卦中不现"。

### P1 — 理论简化（影响分析精度）

3. **进退神只有 7 对**  
   `JIN_SHEN` 只实现寅/申/巳/亥/辰/丑/戌/未，缺少 亥→子/子→亥、巳→午/午→巳 等完整进退。

4. **月令权重简化**  
   当前旺衰用"计分制"，《古筮真诠》要求月令权重优先于日辰，强月旺时日绝"以平论"已实现，但月令合/冲同时存在时的逻辑未完善。

5. **世爻化破特殊规则**  
   自占世爻化破是"假破"，吉凶应看化出的六亲，当前代码没有处理。

### P2 — 功能扩展

6. **可读报告与问事类型耦合**  
   `format_readable_report` 中"七、建议"段硬编码了失物场景语言，其他类型（财运/婚姻/官运）的建议模板缺失。

7. **README 与代码描述矛盾**  
   README 的双视角表格显示"失物：父母(主视角)+妻财"，而代码中 `DUAL_PERSPECTIVE_TABLE` 的主视角是"妻财"。

## 下一步方案

### 阶段一：补齐核心理论缺口（优先）

**1. 三合局介入吉凶判断**
- 在 `jixiong.py` 的 `judge_dong_gua()` 中，检测到三合局后：
  - 合局生世 = 用旺世兴局（吉）
  - 合局克世 = 世爻受伤局（凶）
  - 用神在合局内 = 增强用神力量参与主判断
- 对应知识点：`zhishidianzongjie.md §三合局`

**2. 伏神取用**
- 在 `data.py` 中为每个宫的纯卦建立"飞爻→伏爻"映射
- 在 `jixiong.py` 的 `find_yong_shen_lines()` 中：用神不上卦时查找伏神位置
- 伏神判断规则：伏空凶、伏破凶、伏旺看飞伏关系

**3. 修复进退神完整列表**
- `JIN_SHEN` 补齐完整 14 对
- 对应更新 `TUI_SHEN` 反向映射

### 阶段二：精度提升

**4. 世爻化破假破规则**
- 在 `dongbian.py` 的 `analyze_moving_line()` 中，若动爻是世爻，化破不标记为 `is_useless`，转而检查化出的六亲

**5. 可读报告解耦问事类型**
- 在 `report.py` 的 `format_readable_report()` 中，将"建议"段改为按 `question_type` 分发，为各类型补充对应建议文本

**6. 修复 README 视角描述矛盾**
- 统一为：失物主视角 = 妻财（价值属性），辅助视角 = 父母（物件本相）

### 阶段三：工程健壮性（持续）

**7. 测试覆盖率量化**
- 添加 `pytest-cov`，生成覆盖率报告，目标 ≥ 85%
- 补充三合局吉凶的端到端测试用例

**8. 类型注解**
- 为所有公开 API 添加 `typing` 注解

**9. 性能**（低优先级）
- `_build_hexagram_table()` 在每次 import 时重建，可考虑 `@lru_cache` 或模块级常量
