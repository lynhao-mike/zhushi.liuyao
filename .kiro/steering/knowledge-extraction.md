# 六爻基础知识提取规范

本文档定义从 `liuyao-gushizhenquan_part*.md`、`第十章.md`、`zhishidianzongjie.md` 等知识库文档中提取理论知识并写入代码的标准化流程。

---

## 一、提取目标

将 MD 知识库中的六爻理论规则，转化为：

1. **`data.py` 数据表** — 可查询的常量字典/集合/函数
2. **模块文档头注释** — 写在各 `.py` 文件顶部的规则速查（docstring）
3. **代码逻辑** — 在 `wangshuai.py`、`dongbian.py`、`yingqi.py`、`jixiong.py` 中实现规则判断
4. **输出规范** — 在 `report.py` 中确保生成的中文文本严格遵守术语表述规则

---

## 二、提取流程（6步）

### 步骤1：通读识别知识点分类

阅读 MD 文档，将内容归入以下分类：

| 分类 | 写入位置 | 举例 |
|------|----------|------|
| **静态对照表** | `data.py` 常量 | 六冲表、六合表、三合局、长生十二宫、相刑、相害、星煞 |
| **旺衰判定规则** | `wangshuai.py` | 月破条件、日冲条件、长生帝旺算旺 |
| **动变判定规则** | `dongbian.py` | 回头生/克、化进退、暗动条件 |
| **吉凶判定规则** | `jixiong.py` | 用神取法、卦局模式、有用/无用动爻 |
| **应期公式** | `yingqi.py` | 旺静逢冲、衰静逢值、月破填实 |
| **表述规范** | `report.py` + 各模块docstring | 六冲互冲写法、月破描述方式 |
| **细节层面知识**（暂缓实现） | `data.py` 注释保留 | 六神类象、爻位寓意、纳音 |

### 步骤2：提取静态对照表

从文档中找到所有"对照关系"，转化为 Python 数据结构：

```python
# 模板：
CONSTANT_NAME = {
    "key1": "value1",  # 注释引用原文出处
    "key2": "value2",
}
```

**关键原则：**
- 每个表必须带注释说明出处章节
- 互冲/互合类关系必须双向注册（如 `"巳": "亥", "亥": "巳"`）
- 如原文明确了能量归宿/帝旺等额外信息，必须同步建表

### 步骤3：提取判定规则

将文档中的"条件→结论"句式转为代码逻辑：

```
原文："月支冲爻支，谓之月破，是爻绝对衰败的征兆"
 ↓
代码：if LIU_CHONG.get(month_zhi) == line_zhi: shuai_reasons.append("月破")
```

**关键原则：**
- 区分"吉凶层面"与"细节层面"——吉凶层面的规则写入判定函数，细节层面的暂放注释
- 优先级要明确——如"月令旺衰优先级最高"、"生旺墓绝从于卦理"
- 特殊规则（如"整体趋旺时日绝当平看"）必须作为独立 if 分支处理

### 步骤4：提取表述规范（最重要！）

从文档中提炼术语表述的**正确格式**和**禁止格式**：

```
✓ 正确："巳亥互冲"、"子午互冲"
✗ 禁止："亥冲巳"、"午冲子"（单向写法）

✓ 正确："巳爻受月破"（月令冲爻）
✗ 禁止："亥月冲巳"（单向，且描述重心错误）

✓ 正确："巳酉丑三合出酉金局（能量归酉）"
✗ 禁止："巳酉丑合金局"（缺失帝旺归宿）
```

将这些规范写入：
1. `data.py` 文件头的【规则速查】区块
2. `report.py` 文件头的【输出规范】区块
3. 各模块函数的 docstring 中

### 步骤5：编写/更新辅助函数

为常用查询提供便捷函数：

```python
def is_xiang_xing(zhi_a, zhi_b): ...
def is_liu_hai(zhi_a, zhi_b): ...
def is_ban_he(zhi_a, zhi_b): ...
def get_chang_sheng(wu_xing, di_zhi): ...
```

### 步骤6：运行测试验证

```bash
python -m pytest tests/ -v
```

确保：
- 所有现有测试仍通过（不破坏已有逻辑）
- 新增数据的正确性可被验证
- 报告输出中不含禁止表述

## 三、表述规范速查（提取时必查）

以下规范在提取任何知识点时都必须遵守：

### 3.1 六冲

| 规则 | 说明 |
|------|------|
| 六冲是**互冲**关系 | 子午互冲、丑未互冲、寅申互冲、卯酉互冲、辰戌互冲、巳亥互冲 |
| 代码中可用 `LIU_CHONG[X]` 查对方 | 但**输出文字**必须写"XY互冲"，不可写"X冲Y" |
| 月破 = 月支与爻支互冲 | 描述为"爻受月破"，方向是"月令冲→爻"（爻为被动方） |
| 日冲静爻 = 暗动 | 描述为"{爻支}与{日支}互冲，冲起暗动"，与月破截然不同 |
| 动爻逢日冲 = 冲起不为散 | 描述为"冲起不为散"（力度加强） |

### 3.2 六合

| 规则 | 说明 |
|------|------|
| 月令合爻 = 旺相 | 无"月绊"之说，月合只论旺 |
| 日令合动爻 = 日绊 | 合而不动，延迟之意，不算旺相 |
| 日令合静爻 = 旺相 | 静爻得日合可计旺 |

### 3.3 三合局

| 规则 | 说明 |
|------|------|
| 巳酉丑合酉金局 | 能量归酉（帝旺爻） |
| 申子辰合子水局 | 能量归子 |
| 寅午戌合午火局 | 能量归午 |
| 亥卯未合卯木局 | 能量归卯 |

### 3.4 长生十二宫（吉凶层面精简）

| 记忆口诀 | 含义 |
|----------|------|
| 两墓 | 火墓戌、木墓未（中性） |
| 二绝 | 酉绝寅、子绝巳（衰相） |
| 一生旺 | 土长生申、旺子（旺相） |

**省略规则：**
- 与五行生克重复的→只论生克不论十二宫
- 金长生巳→巳火克金→只论克不论长生
- 辰为水墓→辰土克水→只论克不论墓
- 四土爻本身为库→土爻无墓

### 3.5 相刑（仅细节层面）

- 四土互刑（丑辰未戌两两互刑+自刑）
- 巳申寅三互刑
- 午亥酉三互刑
- 子卯互刑

### 3.6 相害（仅细节层面）

- 寅巳、卯辰、未子、申亥、酉戌、丑午
- 规律："与他合者相冲、与他冲者相合"

## 四、文件组织原则

```
liuyao/
├── data.py          ← 所有静态数据表 + 文件头【规则速查】
├── wangshuai.py     ← 旺衰规则 + 头部规则注释
├── dongbian.py      ← 动变规则 + 暗动规则 + 头部规则注释
├── yingqi.py        ← 应期公式 + 头部规则注释
├── jixiong.py       ← 吉凶判定 + 用神表
├── report.py        ← 输出格式化 + 头部【输出规范】
├── hexagram.py      ← 排卦（纯数据计算，不涉及理论判断）
└── analyzer.py      ← 流程编排（串联以上模块）
```


```
1. 严格遵守六冲互冲表述规范
```
